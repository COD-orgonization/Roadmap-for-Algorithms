Рекурсия — это метод решения задачи, при котором функция вызывает саму себя непосредственно или косвенно. Она состоит из:

- **Базового случая:** Условие, при котором функция завершает работу и возвращает результат без дальнейших вызовов (предотвращает бесконечную рекурсию).
- **Рекурсивного случая:** Часть функции, где она вызывает саму себя с изменёнными (упрощёнными) аргументами, приближаясь к базовому случаю.

# Объяснение для 5 летнего ребёнка:

Представь, что у тебя есть матрёшка (кукла, внутри которой прячутся другие, такие же, но меньше).

Чтобы найти самую маленькую матрёшку, ты делаешь одно и то же действие раз за разом:
1. Ты открываешь матрёшку, которая сейчас в руках.
2. Если внутри есть ещё одна матрёшка, ты достаёшь её.
3. Потом делаешь то же самое с этой, новой, матрёшкой: открываешь и смотришь, есть ли внутри ещё одна.

Так ты будешь повторять одно и то же («открывай и смотри») до тех пор, пока однажды не откроешь матрёшку и не увидишь, что внутри пусто. Это значит, что ты нашёл самую маленькую! На этом игра заканчивается.

**Рекурсия — это как игра с матрёшкой:**
- **Открывать каждую новую матрёшку** — это как вызывать саму себя снова (рекурсивный случай).
- **Найти пустую матрёшку и остановиться** — это базовый случай (условие выхода).
- А весь процесс — это рекурсия.

# **Пример кода на C#:**

```
// Пример: Вычисление факториала
static int Factorial(int n)
{
	// Базовый случай
	if (n == 0)
		return 1;
	// Рекурсивный случай
	return n * Factorial(n - 1);
}
```

```
// Пример: Поиск n-го числа Фибоначчи (неоптимально для больших n)
static int Fibonacci(int n)
{
	// Базовые случаи
	if (n == 0)
		return 0;
	if (n == 1)
		return 1;
	// Рекурсивный случай
	return Fibonacci(n - 1) + Fibonacci(n - 2);
}
```
